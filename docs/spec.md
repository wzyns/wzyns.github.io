# Specification

마크다운 파일 기반의 static site 개인 블로그 프로젝트입니다.

## Concepts

0. 블로그 코드베이스 및 포스트 파일 관리는 최대한 단순해야 한다는 기본 철학으로 합니다.
1. 파일 탐색기 모양으로 포스트를 탐색하는 특이한 디자인을 갖추고 있습니다.
2. 포스트는 마크다운 파일(.md)로 작성하며, posts 디렉터리에 저장됩니다. 실제 디렉터리 구조가 블로그의 파일 탐색기 뷰에 렌더링되어야 합니다.

## Pages

### 1. Home (/)

1. 파일 탐색기 형태의 뷰를 통해 포스트를 탐색할 수 있습니다.

### 2. Post (/[path-to-file])

1. 페이지 상단에는 breadcrumb 경로 바, 파일 이름 아래에는 작성일시가 노출됩니다.
2. 본문에는 마크다운 파일의 내용이 렌더링되어 노출됩니다.

## 파일 탐색기

1. 기본적으로 1-depth로만 디렉터리와 파일을 나열합니다. 디렉터리의 내용물을 확인하려면, 디렉터리를 클릭해 이동합니다. 디렉터리의 path는 url에 반영됩니다.
2. 탐색기에는 디렉터리와 파일이 노출되며, 디렉터리/파일명과 수정일시가 노출됩니다. 정렬 순서는 사전순(alphabetical order)입니다.
3. 파일을 클릭하면, 해당 Post 페이지로 이동합니다.
4. 파일 탐색기의 첫번째 항목은 이전 디렉터리로 이동하기 위한 .. 항목으로 함

## Breadcrumb 경로 바

1. 파일 탐색기와 Post 페이지 상단에 현재 경로를 breadcrumb 형태로 표시합니다.
2. 최상단 디렉터리(posts/)는 뷰 상에서 `/`로 표시됩니다.
3. 각 경로 세그먼트는 클릭 가능한 링크이며, 해당 디렉터리로 이동합니다. 마지막 세그먼트(현재 위치)는 링크가 아닌 텍스트로 표시됩니다.
4. 예: `/` > `dev-log` > `first-entry.md`

## Markdown 파일

1. Frontmatter : 없음. 마크다운 파일명이 포스트의 제목, 마크다운 파일의 생성일시와 수정일시가 포스트의 생성일시와 수정일시가 됩니다. (관리가 단순해야 하기 때문에 이런 정보들을 markdown frontmatter로 관리하지 않습니다)
2. GitHub Flavored Markdown을 사용합니다. (remark-gfm 같은 라이브러리 사용)
