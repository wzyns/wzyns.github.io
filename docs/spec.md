# Specification

마크다운 파일 기반의 static site 개인 블로그 프로젝트입니다.

## Concepts

0. 블로그 코드베이스 및 포스트 파일 관리는 최대한 단순해야 한다는 기본 철학으로 합니다.
1. 파일 탐색기 모양으로 포스트를 탐색하는 특이한 디자인을 갖추고 있습니다.
2. 포스트는 마크다운 파일(.md)로 작성하며, posts 디렉터리에 저장됩니다. 실제 디렉터리 구조가 블로그의 파일 탐색기 뷰에 렌더링되어야 합니다.

## Pages

### 1. Home (/)

1. 파일 탐색기 형태의 뷰를 통해 포스트를 탐색할 수 있습니다.

### 2. Post (/[path-to-file])

1. 페이지 상단에는 breadcrumb 경로 바, 파일 이름 아래에는 작성일시가 노출됩니다.
2. 마크다운 파일(.md)인 경우, 본문에는 마크다운 파일의 내용이 렌더링되어 노출됩니다.
3. 소스코드 파일인 경우, 구문 강조(syntax highlighting)가 적용된 코드 뷰어가 노출됩니다.
4. 본문 하단에 giscus 댓글 위젯이 노출됩니다.
5. 페이지 타이틀: `{파일명} — wzyns`

### 3. Directory (/[path-to-dir])

1. 파일 탐색기 형태의 뷰를 통해 해당 디렉터리의 내용물을 탐색할 수 있습니다.
2. 페이지 타이틀: `{디렉터리명} — wzyns`

## 파일 탐색기

1. 기본적으로 1-depth로만 디렉터리와 파일을 나열합니다. 디렉터리의 내용물을 확인하려면, 디렉터리를 클릭해 이동합니다. 디렉터리의 path는 url에 반영됩니다.
2. 탐색기에는 디렉터리와 파일이 노출되며, 디렉터리/파일명과 수정일시가 노출됩니다. 정렬 순서는 사전순(alphabetical order)입니다.
3. 파일을 클릭하면, 해당 Post 페이지로 이동합니다.
4. 파일 탐색기의 첫번째 항목은 이전 디렉터리로 이동하기 위한 `..` 항목으로 합니다.
5. 이미지 파일(.png, .jpg, .jpeg, .gif, .svg, .webp, .avif, .ico)은 탐색기에 노출되지 않습니다.
6. 파일 타입별 아이콘: 폴더 아이콘, 마크다운 아이콘, 소스코드 아이콘(`<>`), 기본 파일 아이콘(fallback).
7. 수정일시는 클라이언트 시간대 기준으로 표시됩니다.

## Breadcrumb 경로 바

1. 파일 탐색기와 Post 페이지 상단에 현재 경로를 breadcrumb 형태로 표시합니다.
2. 최상단 디렉터리(posts/)는 뷰 상에서 `/`로 표시됩니다.
3. 각 경로 세그먼트는 클릭 가능한 링크이며, 해당 디렉터리로 이동합니다. 마지막 세그먼트(현재 위치)는 링크가 아닌 텍스트로 표시됩니다.
4. 예: `/` > `dev-log` > `first-entry.md`

## Markdown 파일

1. Frontmatter : 없음. 마크다운 파일명이 포스트의 제목, 마크다운 파일의 생성일시와 수정일시가 포스트의 생성일시와 수정일시가 됩니다. (관리가 단순해야 하기 때문에 이런 정보들을 markdown frontmatter로 관리하지 않습니다)
2. GitHub Flavored Markdown을 사용합니다. (remark-gfm)
3. 구문 강조: Shiki (github-dark 테마)
4. 헤딩 앵커: 각 헤딩 우측에 `#` 링크 아이콘이 항상 노출되며, 클릭 시 해당 헤딩의 URL로 이동합니다.
5. 이미지: posts 디렉터리 내 이미지 파일을 상대경로로 참조할 수 있습니다. 빌드 시 이미지가 public/posts/로 복사됩니다.

## 댓글

1. giscus를 사용하여 GitHub Discussions 기반 댓글을 지원합니다.
2. 마크다운 포스트 및 소스코드 뷰어 하단에 노출됩니다.
3. pathname 기반 매핑을 사용합니다.

## SEO

1. `/sitemap.xml`: 빌드 시 정적 생성되는 사이트맵. 홈, 디렉터리, 파일 페이지를 포함하며, 파일 탐색기와 동일한 기준(이미지 제외)을 따릅니다.
2. `/robots.txt`: 사이트맵 위치를 명시합니다.

## 폰트

1. 기본(monospace): Source Code Pro
2. 포스트 본문(sans-serif): Pretendard
3. 코드 영역(`<code>`, `<pre>` 등)은 Source Code Pro를 유지합니다.

## 배포

1. GitHub Actions를 통해 GitHub Pages로 배포합니다.
2. 도메인: wzyns.github.io
